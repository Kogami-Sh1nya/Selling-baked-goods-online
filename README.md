# Bakery Street — онлайн-продажа выпечки

Полноценный учебный репозиторий сайта интернет-магазина выпечки с REST API, PostgreSQL, Express.js, React, Node.js, JWT-авторизацией и HTTPS.

## Использованный стек

**Frontend:** React + Vite + React Router. React выбран для компонентного интерфейса: карточки товаров, формы, модальные окна, корзина и профиль разделены логически. Vite быстрее классического Create React App при запуске и сборке.

**Backend:** Node.js + Express.js. Express используется для REST API: `/api/products`, `/api/orders`, `/api/auth`. Node.js хорошо подходит для JSON API и интеграции с PostgreSQL через `pg`.

**База данных:** PostgreSQL. Используются связи категорий, товаров, пользователей, заказов, позиций заказа, отзывов и тестовых платежей.

**Безопасность:** JWT для авторизации по ролям, HTTPS через локальный self-signed сертификат, строгая валидация email/пароля на клиенте и сервере.

## Аналоги и отличия

- Вместо **React** можно использовать Vue или Angular. Vue проще для небольших проектов, Angular строже и тяжелее, React гибче и популярнее для SPA.
- Вместо **Express** можно использовать NestJS или Fastify. NestJS лучше для больших корпоративных проектов с DI и модулями, Fastify быстрее, Express проще и понятнее для учебного проекта.
- Вместо **PostgreSQL** можно использовать MySQL или MongoDB. PostgreSQL удобнее здесь из-за надежных связей, ограничений и транзакций заказов.
- Вместо **JWT** можно использовать сессии. JWT удобен для REST API и отделения frontend от backend.

## Роли

- **Гость:** просмотр каталога, поиск товара, регистрация и авторизация.
- **Пользователь:** возможности гостя, оформление заказа, тестовая оплата, история заказов, отзывы.
- **Продавец:** просмотр заказов, изменение статуса, управление товарами и ценами через защищённые REST endpoints.
- **Администратор:** полный доступ к заказам, товарам, категориям, отзывам и панели управления.

## REST API

- `POST /api/auth/register` — регистрация.
- `POST /api/auth/login` — авторизация.
- `GET /api/categories` — категории.
- `GET /api/products` — каталог и поиск.
- `GET /api/products/:id` — подробная карточка товара.
- `POST /api/products` — добавить товар, роли seller/admin.
- `PATCH /api/products/:id` — изменить товар, роли seller/admin.
- `DELETE /api/products/:id` — удалить товар, роль admin.
- `POST /api/orders` — оформить заказ, авторизованный пользователь.
- `GET /api/orders/my` — история заказов.
- `GET /api/orders` — список заказов, seller/admin.
- `PATCH /api/orders/:id/status` — смена статуса, seller/admin.
- `POST /api/products/:productId/reviews` — отзыв.

## Установка и запуск

### 1. Требования

Установить Node.js 20+, PostgreSQL 14+ и OpenSSL.

### 2. Создать базу данных

```bash
createdb baking
```

Или через psql:

```sql
CREATE DATABASE baking;
```

### 3. Настроить переменные окружения

```bash
cp .env.example server/.env
```

Откройте `server/.env` и укажите свой пароль PostgreSQL.

### 4. Установить зависимости

```bash
npm run install:all
```

### 5. Создать HTTPS-сертификаты

```bash
cd server
npm run certs
cd ..
```

### 6. Применить SQL-код БД

```bash
cd server
psql -U postgres -d baking -f database/schema.sql
psql -U postgres -d baking -f database/seed.sql
cd ..
```

### 7. Запустить проект

```bash
npm run dev
```

Frontend: `https://localhost:5173`  
Backend: `https://localhost:5000/api/health`

Браузер может показать предупреждение о self-signed сертификате. Для локальной проверки его нужно принять.

## Тестовые аккаунты

Пароль у всех тестовых аккаунтов: `123456789`.

- `admin@bakery.local` — администратор.
- `seller@bakery.local` — продавец.
- `user@bakery.local` — пользователь.

## Проверка функционала

1. Откройте главную страницу и проверьте каталог по категориям: Торты, Пирожные, Печенье, Капкейки, Круассаны, Макаруны, Чизкейки.
2. Введите название товара в поиск.
3. Нажмите на карточку товара — откроется расширенная карточка поверх страницы.
4. Добавьте товар в корзину, проверьте кнопки `-`, количество, `+`.
5. Перейдите в корзину, заполните персональные данные и тестовую оплату.
6. Нажмите `Order now / Оформить заказ`.
7. В профиле проверьте историю заказов.
8. Войдите как продавец или администратор и проверьте блок панели управления.

## Тестовая платежная система

Платёжная система учебная: она принимает любые fake-данные карты формата 16 цифр. После оформления создаётся запись в таблице `payments` со статусом `paid`, затем заказ создаётся транзакционно.
